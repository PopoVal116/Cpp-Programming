# Лабораторная работа №1 — Простые классы и инкапсуляция

## Теория
**Класс** — пользовательский тип данных, объединяющий поля (данные) и методы (функции).

**Основные принципы ООП:**
- Инкапсуляция — скрытие внутренних данных через `private` и доступ через `public` методы (геттеры/сеттеры).
- Конструктор — специальный метод, вызываемый при создании объекта.
- Деструктор — вызывается при уничтожении (не обязателен в простых классах).

**Модификаторы доступа:**
- `private` — доступ только внутри класса
- `public` — доступ извне
- `protected` — будет нужен при наследовании

## Реализовано в коде
- Класс `Drob` — обыкновенная дробь
  - Поля: `chislitel`, `znamenatel` (`private`)
  - Конструктор с проверкой деления на ноль
  - Методы: геттеры/сеттеры, `outputTo()`, операции умножения и деления
- Класс `LibraryReader` — читатель библиотеки
  - Поля: ФИО, профессия, номер билета
  - Демонстрация создания объектов в стеке и в куче (`new`/`delete`)
